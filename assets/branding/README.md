# Branding Assets

This folder is the single source of truth for app icon and splash branding inputs.

## Required source files

- `app_icon.png`
  Master app icon used for iOS, web, macOS, Windows, and the Android legacy launcher icon.
- `app_icon_foreground.png`
  Foreground image for Android adaptive icons.

## Optional density examples

- `2.0x/app_icon.png`
- `3.0x/app_icon.png`

These are sample business-asset density variants. Flutter picks them automatically when
you reference the base asset path in code.

## Configured generation pipeline

The project reads these branding files from `pubspec.yaml`:

```yaml
flutter_launcher_icons:
  image_path: assets/branding/app_icon.png
  adaptive_icon_background: "#F3F7F6"
  adaptive_icon_foreground: assets/branding/app_icon_foreground.png

flutter_native_splash:
  color: "#F3F7F6"
  image: assets/branding/app_icon.png
```

## Commands

Run the full branding pipeline:

```bash
./tool/generate_branding.sh
```

Run individual steps:

```bash
dart run flutter_gen:flutter_gen_command -c pubspec.yaml
dart run flutter_launcher_icons
dart run flutter_native_splash:create
```

## Generated outputs

- Android launcher icons:
  `android/app/src/main/res/mipmap-*`
- Android adaptive icon resources:
  `android/app/src/main/res/mipmap-anydpi-v26`
- iOS app icon set:
  `ios/Runner/Assets.xcassets/AppIcon.appiconset`
- Web icons:
  `web/icons`
- macOS app icon set:
  `macos/Runner/Assets.xcassets/AppIcon.appiconset`
- Windows icon:
  `windows/runner/resources/app_icon.ico`
- Native splash resources:
  Android, iOS, web platform splash files generated by `flutter_native_splash`

## Team rule

When design updates branding:

1. Replace `app_icon.png`
2. Replace `app_icon_foreground.png` if Android adaptive icon foreground changed
3. Run `./tool/generate_branding.sh`
4. Commit both the source branding files and the generated platform outputs
